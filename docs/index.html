    Quick start.

    Configure target repositories once, then run the platform wrapper for export and optional attachment cleanup.

    Windows export

    Copy-Item .env.example .env
notepad .env
powershell -ExecutionPolicy Bypass -File .\run_export.ps1
powershell -ExecutionPolicy Bypass -File .\run_download_attachments.ps1
powershell -ExecutionPolicy Bypass -File .\run_cleanup.ps1

    Bash export

    cp .env.example .env
${EDITOR:-nano} .env
./run_export.sh
test -d .venv || uv venv .venv
uv pip install -r requirements-download.txt
.venv/bin/python -m playwright install chromium
.venv/bin/python download_attachments.py --out-root export --profile-dir export/browser_profile
.venv/bin/python cleanup_img_ext.py --export-root export --debug

    Built for repeatable repo audits.

    Use it before migrations, compliance snapshots, vendor handoffs, or private repository backups.

    Issues and pull requests

    Exports open and closed items with creation and update timestamps.

    Comment history

    Preserves issue comments, PR comments, and review comments in chronological order.

    Local assets

    Rewrites embedded image links to downloaded assets when files are available.

    Offline reruns

    Keeps raw GitHub API responses so Markdown can be regenerated without another fetch.

    Output stays predictable.

    Each repository gets separate issue, PR, asset, and raw folders.
